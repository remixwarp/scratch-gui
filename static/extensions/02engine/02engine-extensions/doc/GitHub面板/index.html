1. 插件 API (ctx.*) - 插件开发推荐调用

插件通过 plugin.init(ctx) 和 plugin.onHook 接收此上下文对象。这些是插件开发时推荐使用的 API，相对稳定。

💡 插件开发最佳实践

在 plugin.init 中缓存 ctx: plugin._ctx = ctx，然后在 Hook 中使用。推荐监听 ui:ready 来扩展 UI。

1.2 GitHub API (ctx.api)

封装 GitHub REST API，自动处理 Token 和缓存

api.putFile owner, repo, path, content, message, branch, sha? Promise<any> 创建/更新文件（content 需 base64）

api.forkRepo owner, repo Promise<any> Fork 仓库

📌 使用建议

  • 插件开发推荐调用：ctx.api.*、ctx.utils.*、ctx.components.*、ctx.manager.*、ctx.core.*（读取状态）
  • 谨慎调用：ctx.extension.*（公共方法相对稳定，私有方法可能变动）
  • 避免直接调用：ctx.extension._xxx（私有方法），除非你接受升级后插件可能失效的风险
  • Hook 扩展：在 onHook 中监听 ui:ready、file:open、dir:load 等事件，避免直接覆盖内部方法
