🐱 WitCatMarkDown 扩展文档

为 Scratch TurboWarp 打造的强大 Markdown 渲染器

📖 简介

WitCatMarkDown 是由白猫原在 CCW 平台开发的 Markdown 渲染扩展，基于改造合并适配 02engine(turbowarp)引擎。它支持部分Markdown 语法，提供了一些功能：

  • 部分 Markdown 支持：标题、列表、代码块、表格、链接、图片等
  • 丰富的扩展语法：颜色、背景、大小、居中、自定义标签、触发器按钮等
  • 交互式编辑器：内置 Markdown 编辑面板，支持文件管理、实时预览
  • 自定义解析规则：可添加、导入/导出、管理正则替换规则
  • 样式与动画：支持元素变换、过渡动画、字体加载等
  • 数学符号优化：自动美化数学符号（π、√、≠等）

🔍 关键积木详细说明

触发器按钮

使用方法

在 Markdown 内容中添加触发器语法：

{trigger=0.5}[按钮文本](myEventID)

参数说明：

  • 0.5 - 冷却时间（秒），防止重复触发
  • 按钮文本 - 按钮显示的内容（可包含 HTML）
  • myEventID - 触发器 ID，用于匹配"当触发器 ID [ID] 被点击"积木

示例代码块：

{trigger=1}[点击我](button1)

然后在 Scratch 中使用：

当触发器 ID [button1] 被点击
自定义ID 为 [button1] 的互交按钮外观html为 [<b>已点击!</b>]

自定义规则管理

功能说明

自定义规则允许你通过正则表达式动态替换 Markdown 内容，实现自定义语法。

积木清单

  • 添加自定义规则 匹配[match] 替换为[replace]
    • match: 正则表达式字符串（如 \\[myTag\\](.*?)\\[/myTag\\]）
    • replace: 替换内容（如 <span style="color:red">$1</span>）
  • 清空自定义规则 - 删除所有规则
  • 导入自定义解析式集json为 [JSON] - 从 JSON 字符串批量导入
    JSON 格式：
    [
 {"match": "\\\\[foo\\\\](.*?)\\\\[/foo\\\\]", "replace": "<b>$1</b>"},
 {"match": "bar", "replace": "baz"}
]
  • 导入自定义解析式集网址为 [URL] - 从网络 URL 加载规则（返回 JSON 数组）

解析顺序管理

使用"打开自定义规则管理面板"积木可调出可视化界面：

  • 拖放调整解析管线中各步骤的顺序
  • 勾选/取消勾选启用或禁用某个步骤
  • 添加、搜索、删除自定义规则
  • 导入/导出规则 JSON 文件
  • 实时测试规则效果

创建与定位

坐标系统

所有位置和大小的单位均为 Scratch 舞台坐标系（像素），范围：

  • X: 0 ~ 舞台宽度
  • Y: 0 ~ 舞台高度

超出范围的值会被自动限制。

创建积木

创建 markdown ID[id] X[x] Y[y] 宽[width] 高[height] 内容[text]

ID 是区域的唯一标识符，后续所有操作都依赖此 ID。

从文件创建

创建 markdown ID[id] X[x] Y[y] 宽[width] 高[height] 从文件[filename]

从内置文件系统加载内容。需先使用编辑器面板创建并保存文件。

元素选择与变换

元素选择语法

许多积木需要指定"第[num]个[type]"，其中：

  • num: 元素序号（从 1 开始）
  • type: HTML 标签名，如 p, img, code, table, a 等

示例：markdown[myText]第[1]个[img]元素缩放X[2]Y[2] 将第一个图片放大 2 倍。

变换积木

  • 偏移: 在原始位置基础上移动
  • 缩放: 设置 scale(X, Y)，1 为原始大小
  • 旋转: rotate(deg)，顺时针角度
  • 3D 变换: 使用 translate3d 和 rotateX/Y/Z

所有变换会自动应用过渡动画（如果设置了过渡时间）。

Markdown 内容


示例 3：自定义规则应用

目标

将 [warning]危险操作[/warning] 转换为红色警告框。

步骤

  1. 使用积木添加规则： 
    添加自定义规则 匹配[\[warning\](.*?)\[/warning\]] 替换为[
<div class="WitCatTag-warning" style="color:red;font-weight:bold">
 ⚠️ $1
</div>]
  2. 在 Markdown 中使用： 
    [warning]请勿删除系统文件！[/warning]

⚠️ 注意事项

性能与兼容性

  • 大量复杂的自定义规则会影响渲染性能，建议保持规则数量在 50 条以内
  • 频繁触发的动画（如每帧变换）可能导致卡顿，使用过渡动画更流畅
  • 数学符号美化会遍历所有文本节点，对超大文本可能产生性能影响
  • 表格自动缩放功能在窗口大小频繁变化时可能触发重排，建议在稳定后使用

安全限制

  • HTML 块中的 JavaScript 代码运行在浏览器环境中，无法直接访问 Scratch 变量
  • 字体加载仅允许来自 https://m.ccw.site, https://m.xiguacity, https://static.xiguacity 的字体文件
  • 从 URL 导入规则时，确保目标域支持 CORS，否则 fetch 会失败

ID 管理

  • 每个 markdown 区域的 ID 必须唯一，否则新创建的区域会替换旧的
  • ID 可以是任意字符串，建议使用有意义的名称（如 mainText, statusPanel）
  • 删除区域后，相关变换和样式设置会自动清除

编辑器面板特性

  • 支持拖拽本地图片到编辑器，自动转换为 Base64 嵌入
  • 支持粘贴剪贴板中的图片（如截图）
  • 文件列表支持重命名、删除、导入/导出
  • 实时预览与编辑区同步滚动
