A*寻路（astarPathfindingTW）扩展文档

适用于 Scratch / TurboWarp 扩展环境。支持：单段路径、必经点（最多5个）、规划算法（平衡/准确/迅速/贪婪/双向）与示意图编辑查看。

网格4方向 路径输出 w/a/s/d 可分步计算 示意图支持 + / - 缩放按钮

1. 扩展概览

本扩展用于在二维网格地图上进行寻路，并可设置起点、终点和必经点。扩展输出的路径是一串方向字符：

  • w：向上（y-1）
  • s：向下（y+1）
  • a：向左（x-1）
  • d：向右（x+1）

坐标系按数组方式：左上角为(0,0)，x向右增大，y向下增大。

2. 地图模型（建图内容 C）

地图由 W（宽）× L（长/高）组成，C 是一段字符串（长度至少为 W*L，短了会把缺失部分当墙）。

  • '0'：可通行
  • 非 '0'（例如 '1'）：障碍/墙

2.1 示例：10×10 全通行

W=10, L=10
C="0000000000".repeat(10)

2.2 示例：带墙

// 5x5 示例（每行5个字符，共5行，最终拼成一段）
00000
01110
01010
01110
00000

你在输入 C 时可以直接把多行拼接成一行字符串；扩展内部按索引读取。

4. 规划算法（ALG）菜单含义

平衡（balanced）

  • 必经点数量 ≤ K：使用“精确”规划（BFS预处理 + DP 旅行商式顺序）
  • 必经点数量 > K：使用“贪心”规划（按曼哈顿距离选下一个点）
  • 单段寻路使用 A* 权重=1.0

准确（accurate）

  • 强制“精确”规划（耗时较高）
  • 单段寻路 A* 权重=1.0

迅速（fast）

  • “贪心”规划
  • 单段寻路 A* 权重=1.5（更快但可能非最短）
  • 可能不是最短路径

贪婪迅速（greedy_fast）

  • “贪心”规划
  • 单段寻路 A* 权重=5.0（很快但更偏贪婪）
  • 可能不是最短路径

双向贪婪迅速（bidirectional_fast）

  • “贪心”规划
  • 单段寻路采用双向搜索（权重=1.5）
  • 可能不是最短路径

提示：当地图大、障碍多、必经点多时，“准确/精确”模式的预处理与DP可能较慢。

5. 示意图窗口使用说明（drawMap）

5.1 工具栏

  • 运行：清缓存后立即计算路径并绘制
  • +：放大（缩放比例 +0.25，上限 10）
  • -：缩小（缩放比例 -0.25，下限 0.2）
  • 重置视图：缩放恢复 100%，平移归零
  • 清除路径：只清缓存与路径显示，不改地图

5.2 查看/编辑模式

  • 仅查看：拖拽画布平移（或选择“移动”工具）
  • 可编辑：可用调色板修改格子、设置起点/终点/必经点

5.3 操作方式

  • 鼠标滚轮：缩放
  • 拖拽：平移或绘制（取决于模式/工具）
  • 触控双指：缩放
  • 窗口标题栏拖拽：移动窗口位置

示意图颜色：绿色为通路，红色为障碍；蓝色方块为起点，紫色方块为终点，橙色圆点为必经点，路径为蓝色线条。

7. 常见问题（FAQ）

7.1 为什么返回 -1？

  • 起点/终点在墙上
  • 无连通路径
  • 计算路程超时（2.5秒仍未完成）

7.2 为什么“计算一步最优解”会返回 none？

表示本次时间片内还没算完，需要下一次调用继续推进。

7.3 必经点最多几个？

最多 5 个（超过会被忽略）。

7.4 “平衡”的 K 值有什么意义？

K 越大越倾向精确（更慢但顺序更优），K 越小越倾向贪心（更快但可能不是最短总路程）。

8. 返回值/状态约定

接口可能返回含义
计算路程 路径字符串 / -1 成功返回完整路径；失败或超时返回 -1
计算一步最优解 w/a/s/d / none / -1 单步指令 / 未完成 / 无解或已结束
估计进度 例如 预处理 20%、规划 55%、构建 90% 用于UI显示
