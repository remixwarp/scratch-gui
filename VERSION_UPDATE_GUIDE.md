# RemixWarp 版本更新日志指南

## 提交信息格式

为了让版本更新日志能够正确显示更新内容，请按照以下格式编写提交信息：

### 基本格式

```
__版本号__
- 更新内容1
- 更新内容2
- 更新内容3
```

### 示例

```
__1.0.0__
- 新增：支持自定义积木返回值
- 优化：提升扩展加载速度
- 修复：解决背景图片显示问题
```

## 版本号规则

1. **手动指定版本号**：使用 `__版本号__` 格式，例如 `__1.0.0__`
2. **自动递增版本号**：如果不指定版本号，系统会自动在上一个版本号的基础上递增第三位数字
   - 例如：上一个版本是 `1.0.0`，新版本会自动变为 `1.0.1`

## 更新类型标记

系统会根据提交信息中的关键词自动识别更新类型：

### ✨ 新增功能 (feature)
- 关键词：新增、添加、new、add、feature
- 示例：`- 新增：AI助手功能`

### ⚡ 优化改进 (improvement)
- 关键词：优化、改进、improve、optimize、enhance
- 示例：`- 优化：提升编辑器加载速度`

### 🐛 问题修复 (bugfix)
- 关键词：修复、bug、fix、issue
- 示例：`- 修复：解决扩展加载失败的问题`

### 📝 其他更新 (other)
- 未匹配到上述关键词的内容
- 示例：`- 更新文档说明`

## 完整示例

### 示例 1：带版本号的提交
```
__3.2.38__
- 新增：版本更新日志功能，自动显示更新内容
- 优化：减少首次加载时间，提升用户体验
- 修复：壁纸显示问题，现在可以正确显示背景图片
- 新增：捐款提示功能，在特定启动次数时显示
```

### 示例 2：不带版本号的提交
```
- 修复扩展库加载错误
- 优化代码分割策略
- 更新依赖包版本
```
系统会自动检测版本号，如果没有找到，会自动递增版本号。

## 注意事项

1. **版本号格式**：使用语义化版本号格式 `主版本号.次版本号.修订号`（如 `1.0.0`）
2. **更新内容**：每行以 `-` 开头，清晰描述更新内容
3. **简洁明了**：每条更新内容尽量简洁，突出重点
4. **分类清晰**：系统会自动根据关键词分类，请使用正确的关键词

## 用户界面说明

### 更新日志窗口

- **自动弹出**：当检测到新版本时，窗口会自动弹出并置于最上层
- **版本信息**：显示当前版本号和更新日期
- **更新内容**：按类型分类显示，带有图标标识
- **不再显示**：用户可以选择不再显示此版本的更新日志
- **响应式设计**：适配不同屏幕尺寸，包括移动端

### 存储机制

- 使用 `localStorage` 存储用户偏好设置
- `remixwarp_last_seen_version`：记录上次查看的版本号
- `remixwarp_dont_show_updates`：记录用户是否选择不再显示
- `remixwarp_version_history`：存储版本历史记录（最多20条）
